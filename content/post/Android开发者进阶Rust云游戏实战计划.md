### 第二阶段: 逻辑下沉 - "高性能网络编程"

**目标**：将有状态的、对性能敏感的逻辑交给 Rust 处理。

#### 2.1 Rust网络编程入门

* **核心库**: `std::net::UdpSocket`(标准库)或者`socket2`(更底层)
* **任务**：写一个简单的 Rust 命令行程序，给手机发 UDP 包。

#### 2.2 进阶实战："网络质量探测器(Heartbeat Module)"

* 背景: 云游戏需要精确知道当前的 RTT 和丢包率。
* 设计思路：
  * Rust 端: 启动一个后台线程(使用`std::thread`或`tokio`)。
  * **逻辑**: 每秒发送一个"Ping"包，等待"Pong",计算时间差。
  * **JNI 接口**: 暴露一个`getNetworkStats()`方法，返回`{ rtt: 50, loss: 0.1 }`给 kotlin。
* 学习并发编程，学习状态管理，有实用价值

### 第三阶段: 工具链赋能 - "效率倍增器"(长期进行)

**目标：** 用 Rust 编写开发工具，让 Android 开发更轻松。

### 3.1 实战项目："伪云游戏服务端 (Fake Cloud Server)"

- **痛点**：开发客户端时，如果依赖真实服务端，经常因为服务端挂了、网络不好或者联调麻烦而浪费时间。
- **任务**：用 Rust 写一个在 Mac/PC 上跑的命令行工具 (CLI)。
  - 它监听 WebSocket 端口。
  - 它能接收 Android 发来的信令握手 (Handshake)。
  - 它能回复一个假的 SDP，甚至发几个假的视频包。
- **核心库**：`clap` (写命令行参数的神器), `tungstenite` (WebSocket 库)。
- **价值**：
  - **解耦**：让客户端开发不再受制于服务端。
  - **跨界**：Rust 写 CLI 工具体验极佳，以后您可以自己写工具处理日志、分析数据。

## 学习资源推荐

### 书籍 (圣经三部曲)

1. **The Rust Programming Language (Rust 程序设计语言)**：官方书，必读前 10 章。中文版翻译很好。
2. **Rust in Action (Rust 实战)**：非常适合想了解系统级编程（内存、二进制、网络）的开发者。
3. **Android NDK Guides**：Google 官方文档，虽然是 C++ 的，但 JNI 的原理是一样的，必看。

### 重点关注的 Crates (第三方库)

- `jni`：Rust 与 Java 交互的桥梁。
- `bytes`：高效处理字节缓冲区（处理视频流/网络包必用）。
- `serde` + `serde_json`：**地表最强** JSON 处理库。用它来解析信令，体验比 Gson/Moshi 好十倍。
- `tokio`：Rust 异步编程的事实标准（网络编程必用）。

**保持 "Sidecar (边车)" 模式**：不要试图重写整个 App。UI 还是用 Kotlin 写，Rust 只做它擅长的脏活累活。

### 终极目标

这份计划的**最终理想状态**，是成为一个**“全栈音视频架构师”**，而不仅仅是一个“会写 Rust 的 Android 开发”。

具体来说，达到理想状态后，您的能力模型和工作流会变成这样：

### 1. 核心资产的“一次编写，到处运行”

- **理想状态**：您手头会有一套自己积累的 **Rust Core Library**。
  - 里面包含了：云游戏信令协议、输入预测算法、网络质量探测、日志上报、甚至是一套精简的音视频播放逻辑。
  - **威力**：当老板说“我们要出 iOS 版”或者“要做个 PC 客户端”时，您不需要重写核心逻辑。您只需要写一层薄薄的 UI (SwiftUI / Tauri / Compose)，底层直接复用这套 Rust Core。
  - **壁垒**：这套 Core 是您的核心竞争力，别人很难维护，只有您懂。

### 2. 对“黑盒”的完全掌控

- **理想状态**：WebRTC 和 FFmpeg 对您来说不再是不可触碰的“黑盒”。
  - **现状**：现在遇到黑屏、卡顿，可能只能猜是网络问题还是解码问题。
  - **未来**：您能在 Rust 层拦截每一个 RTP 包，解析每一帧 H.264 数据。您能精确地告诉后端：“第 5003 号包丢了，导致关键帧缺失，所以花屏了”，甚至自己写逻辑来补救（比如请求重发关键帧）。
  - **威力**：从“调包侠”变成“造轮子的人”，解决疑难杂症的能力指数级上升。

### 3. 极高的工程效率与质量

- **理想状态**：利用 Rust 的特性消灭一整类 Bug。
  - **内存安全**：再也不会有 Native Crash (SIGSEGV)。Rust 编译器帮您挡住了所有空指针和内存泄漏。
  - **并发安全**：再也不会有“多线程竞争导致的数据错乱”。Rust 逼着您写出线程安全的代码。
  - **工具链**：您会习惯用 Rust 写各种 CLI 小工具来辅助开发（比如自动化测试脚本、协议分析工具），开发效率远超纯手写 Java/Kotlin。
