## 定位：Rust + Android高性能开发

```
核心竞争力：
用Rust解决Android的性能瓶颈问题

## 阶段2:HTTP客户端项目(Month 2-3)

```
// 1. 连接池

* TLS握手过程

* Keep-Alive

#### 输出物：

* GitHub开源项目

## 阶段3:图像处理入门(Month 4)

目标：轻度接触多媒体，不深入

为什么选择图像而不是视频？

* 学习曲线平缓
* 应用场景广(图片压缩/滤镜)
* 不需要深入编解码原理
* Rust生态成熟(`image`库)

项目：图片处理工具包

学习内容：

* 图片格式（JPEG/PNG/WebP）基础知识
* 色彩空间（RGB/YUV）概念
* 压缩算法原理（了解即可）
* 并行处理(Rayon)

该阶段暂不需要学：

* 视频编解码
* FFmpeg
* H.264/H.265细节
* 音视频处理

## 实行建议

### 1. 以兴趣为驱动

```
Rust兴趣高 → 让Rust成为主角
选择Rust能解决的问题(网络/存储/图像)
不强求音视频（除非后面感兴趣）
```

### 2. 项目导向学习

```
每个阶段都做一个完整项目:
├─ Month 2-3: HTTP客户端
├─ Month 4: 图片处理
├─ Month 5: 存储引擎
└─ Month 6: 整合App

不要只学理论!
```

### 3. 建立影响力

```
GitHub:
├─ 每个项目都开源
├─ 写清楚README
└─ 目标:至少一个500+ stars项目

博客:
├─ 记录学习过程
├─ 性能对比数据
└─ 发到掘金/Rust中文社区

社区:
├─ 参与Rust中文社区
├─ 回答Rust相关问题
└─ 认识同行
```

### 4. 保持灵活

```
如果在学习过程中:
├─ 发现对音视频产生兴趣 → 深入
├─ 发现对构建工具感兴趣 → 转向
├─ 发现Rust不适合你 → 及时调整

不要死守计划,根据实际感受调整!
```

## 平衡

### 策略1: 工作中找机会(20%学习时间)

关键：把Rust融入工作，而不是对立

#### 场景A：性能优化机会

**怎么做**

```
1. 先用现有方案完成任务(满足业务需求)
2. 周末用Rust写一个优化版本
3. 做性能对比测试
4. 写文档提给team leader

结果:
✅ 工作任务完成
✅ Rust有了实战场景
✅ 展示技术能力(可能加薪/晋升)
```

#### 场景B：重复性工作自动化

```
工作中的重复劳动:
├─ 手动处理资源文件
├─ 批量修改代码格式
├─ 生成重复的UI代码
└─ 测试数据准备

Rust机会:
用Rust写工具脚本自动化这些任务
```

#### 场景C：利用工作中的等待时间

```
工作中的零碎时间:
├─ 代码review等待(10-20分钟)
├─ 编译等待(5-10分钟)
├─ 会议开始前(5-15分钟)
└─ 午休后的空闲(15-30分钟)

利用方式:
├─ 看Rust文档/博客(不需要写代码)
├─ Code Review Rust项目
├─ 刷Rust练习题(Rustlings)
└─ 思考技术方案
```

### 策略2: 下班后碎片化学习

❌ 错误方式: 每天都想写大项目 → 状态进入慢 → 浪费时间 → 产出少

✅ 正确方式: 周一到周五只做"轻量级学习"

**工作日轻量级学习**：

```
// 周一: 读代码(30分钟准备 + 1小时阅读)
- 找一个Rust项目(如tokio/hyper)
- 读一个模块的代码

// 周二: 做练习题(直接开始)
- Rustlings练习题
- LeetCode用Rust实现

// 周三: 看教程/文档

// 周五: 总结 + 规划周末
- 回顾本周学习
- 整理笔记
- 规划周末项目

**周末深度学习**

```
✅ 适合周末做的事:
├─ 完整的项目开发(HTTP客户端)

├─ 架构设计(FFI接口设计)

## 心态管理

### 问题1: "工作都在做UI,学的Rust用不上,怎么办?"

**答案：这是正常的，分离思考**

```
工作:
├─ 赚钱养活自己
├─ 积累Android经验
└─ 保持稳定收入

学习:
├─ 为未来投资
├─ 换工作的筹码
└─ 技术兴趣满足

这是两条并行线:
工作给你"当下",学习给你"未来"
```

### 问题2: “下班太累，想躺平怎么办？”

**答案：接受疲惫，调整预期**

```
累的时候:
✅ 做轻量级学习(读代码/看教程)
✅ 运动30分钟(恢复精力)
✅ 早点睡,第二天早起学

不要:
❌ 强迫自己写代码(效率低)
❌ 内疚自责(负面情绪)
❌ 放弃计划(间歇性努力)

关键: 
"每天进步一点点" > "某天学10小时然后放弃"
```

### 碎片化时间高效利用

```
通勤(地铁/公交):
├─ 听Rust播客/视频
├─ 看Rust文档(手机)
├─ Code Review(GitHub移动端)
└─ 思考技术方案

午休:
├─ 看一篇技术博客(15分钟)
├─ 做几道Rustlings(20分钟)
└─ 不写复杂代码(环境不好)

等待时间:
├─ 编译等待 → 看Rust文档
├─ CI等待 → 读开源代码
└─ 会议前 → 刷练习题
```

### 核心原则

```
1. 工作是工作,学习是学习
   - 不要混淆,分清边界
   
2. 接受现实,长期主义
   - 工作日累了就少学点
   - 重要的是持续性
   
3. 碎片+集中 结合
   - 工作日:轻量级学习
   - 周末:深度项目开发
   
4. 不要burn out
   - 保持生活平衡
   - 运动/社交/休息都重要
   
5. 量化进度,庆祝小胜利
   - 每周复盘
   - 每月庆祝
```

目标：让Rust成为日常习惯，而不是负担。
